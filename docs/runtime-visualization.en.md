# Visualization Subsystem Reference

> Documents the OmegaWiki visualization system: file locations, generation commands, customization, and Obsidian integration.

---

## Two visualization paths

ΩmegaWiki separates visualization into two paths with very different cost and refresh semantics:

### Path A — Obsidian native graph (zero config, no skill call)

Open `wiki/` as an Obsidian vault. Obsidian's built-in graph view auto-parses `[[wikilinks]]` from every page and applies the color groups defined in `wiki/.obsidian/graph.json` (committed to git). Every page added by `/ingest` shows up live; the user does not need to run `/visualize` for this view to stay current.

The only time `/visualize --obsidian` needs to run is when `config/visualize.json` is edited to change the palette — that updates `wiki/.obsidian/graph.json` to reflect the new colors. Day-to-day ingest does not change the palette and therefore does not need this regeneration.

### Path B — Canvas + HTML artifacts (regenerated when graph changes)

Two artifacts read `wiki/graph/edges.jsonl` + `citations.jsonl` at generation time and stale as soon as those files change:

- `wiki/canvases/*.canvas` — Obsidian Canvas with labeled typed edges (`builds_on`, `introduces_concept`, `supports`, …) that Path A cannot show
- `wiki/graph-view.html` — Cytoscape.js interactive explorer with filtering and BFS

Refresh triggers in this codebase:

| Trigger | Behavior |
|---|---|
| `/init` Step 6 fan-in | Auto-runs `generate-canvas` + `generate-html` (best-effort; failure does not fail `/init`). `--obsidian` is **not** rerun — it is static config already in git. |
| `/ingest <source> --visualize` | Opt-in. After Step 7 rebuild, regenerates Canvas + HTML. Skipped automatically in INIT MODE so sibling worktrees don't race. |
| `/visualize [--canvas] [--html] [--focus <node>]` | Manual, on-demand, with full control over scope. |

`/ingest` without `--visualize` does not touch Path B artifacts — they will lag the wiki until the next refresh. This is intentional: Path A already gives the user a live view, and avoiding per-ingest regeneration keeps batch ingestion fast.

## Overview

The visualization subsystem generates three types of artifacts from the wiki's knowledge graph:

1. **Obsidian graph config** — color-coded entity types in Obsidian's built-in graph view
2. **Obsidian Canvas files** — curated knowledge map views with labeled typed edges
3. **Standalone HTML graph explorer** — interactive web-based graph using Cytoscape.js

All artifacts are generated by `tools/visualize.py` and invoked via the `/visualize` skill, or auto-fired by `/init` (Canvas + HTML only) and opt-in via `/ingest --visualize`.

## File Locations

| File | Type | Git | Description |
|------|------|-----|-------------|
| `config/visualize.json` | Config | Yes | Color palette and visualization preferences (hand-maintained) |
| `tools/visualize.py` | Tool | Yes | Python CLI for generating all visualization artifacts |
| `.claude/skills/visualize/SKILL.md` | Skill | Yes | `/visualize` skill definition |
| `wiki/.obsidian/graph.json` | Generated | Yes | Obsidian graph color groups |
| `wiki/.obsidian/app.json` | Generated | Yes | Obsidian app settings (create-only) |
| `wiki/canvases/*.canvas` | Generated | Yes | Curated Canvas views |
| `wiki/graph-view.html` | Generated | Yes | Standalone HTML graph explorer |

## Generation Commands

```bash
# All artifacts at once
python3 tools/visualize.py generate-obsidian-config wiki/
python3 tools/visualize.py generate-canvas wiki/
python3 tools/visualize.py generate-html wiki/
python3 tools/visualize.py list-recommendations

# Focused canvas
python3 tools/visualize.py generate-canvas wiki/ --focus claims/my-claim --depth 2

# Custom output path
python3 tools/visualize.py generate-canvas wiki/ --output /tmp/my-canvas.canvas
python3 tools/visualize.py generate-html wiki/ --output /tmp/graph.html
```

## Color Palette Customization

Edit `config/visualize.json` to change entity type colors:

```json
{
  "entity_colors": {
    "papers":       { "hex": "#4A90D9", "rgb": 4886745 },
    "concepts":     { "hex": "#7B68EE", "rgb": 8087278 }
  }
}
```

- `hex` — used by the HTML graph explorer (Cytoscape.js)
- `rgb` — decimal integer used by Obsidian's `graph.json` (e.g., `0x4A90D9` = 4886745)

After editing, regenerate: `python3 tools/visualize.py generate-obsidian-config wiki/`

## Obsidian Setup

1. Open Obsidian > "Open another vault" > "Open folder as vault"
2. Select the `wiki/` directory
3. The `.obsidian/graph.json` config applies automatically
4. Open Graph View (`Ctrl+G` / `Cmd+G`) to see color-coded entity types

| Dataview | SQL-like queries over frontmatter | Settings > Community Plugins > Browse |
| Excalidraw | Annotate Canvas views | Settings > Community Plugins > Browse |

## Canvas Format

Obsidian Canvas files (`.canvas`) are JSON with this structure:

```json
{
  "nodes": [
    {
      "id": "papers/my-paper",
      "type": "file",
      "file": "papers/my-paper.md",
      "x": 0, "y": 0,
      "width": 400, "height": 300,
      "color": "4"
    }
  ],
  "edges": [
    {
      "id": "edge-0",
      "fromNode": "papers/my-paper",
      "toNode": "concepts/my-concept",
      "fromSide": "right",
      "toSide": "left",
      "label": "introduces_concept"
    }
  ]
}
```

Canvas color indices: "1"=red, "2"=yellow, "3"=orange, "4"=blue, "5"=green, "6"=purple.

## HTML Graph Explorer

The standalone HTML file (`wiki/graph-view.html`) uses Cytoscape.js loaded from CDN.

### Serving

JSONL files require HTTP (not `file://` due to CORS):

```bash
python3 -m http.server 8080 --directory wiki/
# Open: http://localhost:8080/graph-view.html
```

### Features

- Color-coded nodes by entity type
- Directed edges with type labels
- Entity type and edge type filter checkboxes
- Search box (fuzzy match against node titles)
- Click node to highlight BFS neighborhood
- Double-click node to open in Obsidian (via `obsidian://` URI)
- Adjustable BFS depth (1-5)

## Edge Type Visualization

Edges are colored by workflow category:

| Workflow | Color | Edge Types |
|----------|-------|------------|
| ingest | Blue | same_problem_as, similar_method_to, builds_on, introduces_concept, etc. |
| claim_evidence | Green | supports, contradicts |
| experiment | Orange | tested_by, invalidates |
| idea | Yellow | addresses_gap, inspired_by |
| provenance | Gray | derived_from |
